When you create a page with double colons (::) as a separator in the page name, you are using a namespace to define parent and child pages. Sub-namespaces are created by adding more than one :: separator to the page name. Each namespace or sub-namespace can have as many child pages under it as you like (within the resource limits of your hardware). The full path to each page is displayed in the navigation bar with links to its namespace and sub-namespace(s) in the title.

For example, let's say you have these pages:
  • [[Recipes::Desserts]]
  • [[Recipes::Desserts::Cakes]]
  • [[Recipes::Main dishes]]
  • [[Recipes::Side dishes]]
  • [[Recipes::Soups]]
Recipes:: is a namespace. Desserts:: is a sub-namespace of the Recipes:: namespace. Cakes:: is a sub-namespace of the Desserts:: sub-namespace.

If you open the [[Recipes::Desserts::Cakes]] page, the title in the navigation bar contains links to Recipes and Desserts. The Recipes link opens a page listing all pages in the Recipes:: namespace, and the Desserts link opens a page listing all pages in the Desserts:: sub-namespace.

Namespace sub-menu

All pages in any namespace can be given a custom sub-menu by creating a ::Menu page inside the namespace. The sub-menu content will replace the ::Menu content while any pages belonging to that page's namespace are open. The sub-menu will vanish as soon as you navigate away from its namespace.

You can control what the sub-menu displays by editing the page with ::Menu in its name, or by clicking the menu edit icon (bottom right corner of the menu) while it is being displayed. A menu is just a wiki page that is displayed in the menu area, so anything a normal page can contain a menu page can as well; this includes Javascript scripts and CSS style sheets, allowing very flexible customization of namespaces. To include the main menu in a sub-menu use [[Include::::Menu]].

For example, if you create the [[Recipes::Menu]] page with My favorite recipes! as its content, then whenever you open a page in the Recipes:: namespace, a sub-menu will appear with My favorite recipes! in it.

Sub-namespace sub-menu

All pages in any sub-namespace can be given a custom sub-menu by creating a ::Menu page inside the sub-namespace. The sub-menu will be displayed beneath the main menu while any pages belonging to that page's sub-namespace are open. You can determine what goes into the sub-menu by editing the page with ::Menu in its name. The sub-menu will vanish as soon as you navigate away from its sub-namespace.

To include the main menu in a sub-menu use [[Include::::Menu]]. You can also include menus from lower namespaces in the same way.

For example, if you create the [[Recipes::Desserts::Menu]] page with My favorite desserts! as its content, then whenever you open a page in the Recipes::Desserts:: sub-namespace, a sub-menu will appear with My favorite desserts! in it.

If a namespace contains namespace and sub-namespace ::Menu pages and you open a page in the sub-namespace, the contents of the sub-namespace ::Menu page will be displayed. If a sub-namespace has no menu but a lower namespace does, then that menu will be used instead.
NOTE: Sub-menus work the same way menus do. For more information on menus see the Edit the menu page.
There is no limit to how many sub-namespaces you can have, other than the 256 character limit for title length.

Reserved namespaces

Some namespaces are reserved by the wiki for its own internal use or for specific use by the user
  • ::Menu is a special reserved namespace used for the menu and sub-menus
These are listable namespaces:
  • File:: for embedded files
  • Image:: for embedded images
  • Locked:: for listing encrypted pages
  • Special:: for all the special pages
  • Tagged:: for listing tagged pages
  • Unlocked:: for listing plain (unencrypted) pages
  • Untagged:: for listing untagged pages
  • WoaS:: for core WoaS pages
Some namespaces cannot be listed but are used inside wiki links to accomplish a task:
  • Include:: for transclusion
  • Javascript:: for direct Javascript execution
  • Lock:: used internally or in a link to lock (encrypt) a page
  • Tag:: syntax used to tag a page
  • Tags:: alternative syntax used to tag a page
  • Unlock:: used internally or in a link to unlock (decrypt) a page
