Adding Javascript to the wiki source

You can add <script> tags to your wiki source and they will be correctly rendered in the document <head> tag each time the page is loaded; external scripts (using the src attribute) will also be parsed and activated. The order of declaration of the <script> tags will be respected.
NOTE: Since the Javascript tags are activated from within the <head> tag, you cannot use document.write() or similar inline Javascript statements.
It is very often better to use a macro or other Javascript loaded by a custom plugin. The code or macro is then loaded once when the file is first loaded and is available to all your pages.

Adding Javascript to your page

To add Javascript to your page, you must enclose it inside script tags.

Use this code in a page, replacing YOUR_CODE_HERE with your Javascript code:
<script type="text/javascript">
YOUR_CODE_HERE
</script>
To add Javascript to your page.

Using external Javascript

You can include external Javascript on a page by placing the script file in the same directory as the WoaS file and then inserting a reference to it in the body of the page.

Use this code in a page, replacing SCRIPTNAME.JS with the name of your script:
<script src="SCRIPTNAME.JS" type="text/javascript"></script>
To include an external Javascript in a page.

Using the Javascript:: namespace in a link

Using a Javascript expression contained in a link, WoaS can dynamically create a temporary page that only exists while it is being displayed and is not stored as a page in the wiki. It does this through the Javascript:: namespace. Everything following the namespace declaration will be evaluated as a Javascript expression. You can use $JS:: instead of Javascript:: if your Aliases page still contains the default $JS Javascript declaration.
NOTE: Javascript:: links cannot be transcluded, cannot contain double-quotes ("), and do not create history (back/forward entries).
If the Javascript expression returns a non-empty string it will be used as the body of the dynamic page and the link text will be used as the title of the page. If the expression returns an array and that array contains exactly two strings, then the first string will be used as the dynamic page's title (with Javascript:: added), and the second one will form the content of the page. Any other return types will be ignored, as will an empty content string. The content string can be WoaS wiki syntax, HTML, or a combination of both; it is parsed just as a normal page is.

A simple (but useless) example is:
[[Javascript::'=Hi']]
This link should create a dynamic page in the main wiki: Javascript::'=hi'. Note that the Javascript expression is used as the page title; usually not very useful! Another simple example (and equally useless, except we don't have the title this time; note the need for a space after the array's closing bracket):
[[Javascript::['','=Hi'] ]]
And here's the link: Javascript::['','=Hi'] . The space isn't needed if, as is usual, you are using a label after the page link; for example:
[[Javascript::['','Hello World']|A dynamic page]]
These links should open in the main wiki window; the first uses Javascript:: in the page source, the second $JS:: (though you won't see this when hovering over the link if your $JS Alias is set): A dynamic page, A dynamic page.

These simple examples could even contain a link (note the use of ]\] in the link):
[[Javascript::['Hello world','I am Hello World, not the [[Special::All Pages]\] page']|A pointless page]]
The above example as a pointless page link.

Here is a slightly more useful example that uses a Javascript function:
<script type="text/javascript">
  function rhyme1() {
    return ['A Rhyme',
      '=Hello Lamppost\nWhat cha knowing?\n' +
      'I\'ve come to watch\n* your flowers\n* growing.'];
  }
</script>
The link [[Javascript::rhyme1()|Rhymes]] will open this page in the main wiki window: Rhymes

The Javascript function can also be passed dynamic content using one or more function parameters:
<script type="text/javascript">
  function rhyme2(thing) {
    return ['A '+thing+' Rhyme','=Hello '+thing+
      '\nWhat cha knowing?\n' +
      'I\'ve come to watch\n* your flowers\n* growing.'];
  }
</script>
The link [[Javascript::rhyme2('Lamppost')|Rhymes]] will open this page in the main wiki window: Rhymes. This one uses $JS: Another rhyme.

Not that the above example would be more easily accomplished by transcluding a template page and passing in replacement parameters. Nevertheless, there are times when generating changable, dynamic text is perfectly suited to the Javascript:: namespace technique.
