The Special::Import page provides an interface for importing the contents of a current or earlier version of the wiki from an .html source file (back to v0.9.6B). Certain options are only available if the context allows.

Source

Click the Browse... button to select the wiki file to import.

Import configuration (except WSIF datasource settings)

Imports all the settings from the Special::Options page in the source file into the destination file.
NOTE: The settings in the WSIF settings section of the Special::Options page will not be imported.

Import content

Imports these pages from the source file:
  • Any user-created pages.
  • Main Page
  • ::Menu
and also these pages unless Do not import WoaS:: namespace pages is selected:
  • WoaS::Aliases
  • WoaS::Hotkeys
  • WoaS::Plugins

Import custom CSS

Imports the WoaS::CSS::Custom page from the source file, overwriting the same page in the destination file. CSS rules from older releases of the wiki may not work properly in the current version; see Import CSS rules.

Importing custom CSS and configuration from a current WoaS version, while not importing content, is a good way to copy a custom wiki set-up.

Existing pages

Erase all user pages before import process

Erases all user-created pages before importing content into the destination file. The import interface will pop up two confirmation boxes to verify that you're sure you want to erase the pages.

Ignore

Prevents the import interface from overwriting existing pages, allowing only new pages to be imported into the destination file.

Overwrite

Gives the import interface blanket permission to overwrite existing pages in the destination file.

Ask before overwrite

Forces the import interface to ask your permission to overwrite individual pages in the destination file.

Security

Comment Javascript tags

Inserts disabled_ into any script tags in the pages that are imported into the destination file so that scripts will not run.

Example script tag:
<script type="text/javascript">alert("This is an example alert!");</script>
Example disabled script tag:
<disabled_script type="text/javascript">alert("This is an example alert!");</disabled_script>

Comment macro definition/call

Inserts Macro disabled into any macro definitions or macro calls in the pages that are imported into the destination file so that macros will not run.

Example macro call:
<<<woas.include: My Page>>>
Example disabled macro call:
<<< Macro disabled
woas.include: My Page>>>

Do not import WoaS:: namespace pages (aliases, hotkeys, and plugins)

Does not allow import of pages listed.
