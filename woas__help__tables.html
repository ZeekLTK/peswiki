Table syntax

Use this code in a page:

{|
| CELL1 || CELL2
| CELL1 || CELL2
|}

To create this:
CELL1CELL2
CELL1CELL2

Or use this code in a page:

{|
| CELL1 || CELL2
|-
| CELL1 || CELL2
|}

To create this:
CELL1CELL2
CELL1CELL2

The line that starts with |- is a comment line. You can type some text after it which will only be visible when viewing the source:

Use this code in a page:

{|
| CELL1 || CELL2
|- This is a comment.
| CELL1 || CELL2
|}

To create this:
CELL1CELL2
CELL1CELL2

Header rows

Tables can have a specially formatted header row created by adding a pipe followed by a plus symbol and a space before the title text (example: |+ Title), and can contain internal and external wiki links.

Use this code in a page:

{|
|+ TITLE
| *HEADER1* || *HEADER2* || *HEADER3*
|-
| CELL1 || CELL2 || CELL3
|-
| CELL1 || CELL2 || CELL3
|-
| CELL1 || CELL2 || CELL3
|-
| CELL1 || || CELL3
|}

Extended customization of layout

Styling, formatting, and alignment can be used to control the look and behavior of tables. All example tables on this page use table comment lines ( |- ) to make it easier to see what is involved in creating the tables. These lines are not necessary when creating your tables.

Add some custom CSS rules to the wiki

To create a custom table defined by CSS, add some table rules to the wiki CSS using a class name of your choosing.
  • Use these CSS rules in your wiki, replacing CLASS_NAME with the class name you'd like to use, and CSS_RULES with the rules you'd like to define for the table, the caption, the headers, the cells, and a special cell:

    table.CLASS_NAME {CSS_RULES}
caption.CLASS_NAME {CSS_RULES}
table.CLASS_NAME th {CSS_RULES}
table.CLASS_NAME td {CSS_RULES}
table.CLASS_NAME td.special {CSS_RULES}

    See the custom CSS rules used by the examples on this page.

Assign a class to a table

When custom CSS rules have been added to the wiki, you can replace the default {| table opening line with a custom one to assign the custom class to the table.

Open a table with this line of code:

{| class="CLASS_NAME"

To use a custom class for the table.

Example default table:
Use this code to create the table:

Use this code to create the table:

cell 1cell 2cell 3
{| class="mycustomtable"

Alignment

Tables are aligned to the left by default, but alignment can be controlled by surrounding the cell contents with extra spaces or tabs. Forcing content to the left is not necessary, but it may help to use forced alignment for debugging complex tables.

AlignmentExample resultEffect
{space}text{space}textdefault
{space}text{space}{space}texttext is aligned to the left
{space}{space}text{space}texttext is aligned to the right
{space}{space}text{space}{space}texttext is centered

Headers

In default tables, table header cells (<th>) are created by surrounding the cell content in asterisks. In custom tables, use a single = symbol before the cell content to change the table data cell (<td>) into a table header cell <th>, which can be styled differently through CSS.

Example default table:
Use this code to create the table:
cell 1cell 2cell 3
cell 1cell 2cell 3
cell 1cell 2cell 3
{|
|-
| *cell 1* || *cell 2* || *cell 3*
|-
| cell 1 || cell 2 || cell 3
|-
| cell 1 || cell 2 || cell 3
|-
|}
Example custom table:
Use this code to create the table:
cell 1cell 2cell 3
cell 1cell 2cell 3
cell 1cell 2cell 3
{| class="mycustomtable"
|-
| =cell 1 || =cell 2 || =cell 3
|-
| cell 1 || cell 2 || cell 3
|-
| cell 1 || cell 2 || cell 3
|-
|}

Placement of table header cells (<th>) can be anywhere in a table.

Example default table:
Use this code to create the table:

cell 1cell 2cell 3
cell 1cell 2cell 3
{|

| *cell 1* || cell 2 || cell 3
|-
| cell 1 || cell 2 || cell 3
|-
| cell 1 || cell 2 || *cell 3*
|-
|}
Example custom table:
Use this code to create the table:

cell 1cell 2cell 3
cell 1cell 2cell 3
{| class="mycustomtable"
|-
| =cell 1 || cell 2 || cell 3
|-
| cell 1 || cell 2 || cell 3
|-
| cell 1 || cell 2 || =cell 3
|-
|}

Styling table cells

Definitions can be used within tables to style the cells. A definition affects only the cells following it. If you wish to apply the definition to the entire table, insert it directly beneath the table opening tag.

Method 1 - define styles in the wiki CSS

This method requires that you add a custom "special" CSS rule to the wiki so that any table cell in the "special" class will use that rule. See the table td.special rule in the custom CSS used by this page for an example.

You can add a "special" class definition by using |$ class="special" in a table. The definition can be used by any row below the definition by adding == before the contents of a cell.

Example default table:
Use this code to create the table:
cell 1cell 2cell 3
cell 1cell 2cell 3
cell 1cell 2cell 3
{|
|-
|$ class="special"
|-
| *cell 1* || *cell 2* || *cell 3*
|-
| ==cell 1 || cell 2 || ==cell 3
|-
| cell 1 || ==cell 2 || cell 3
|-
|}
Example custom table:
Use this code to create the table:
cell 1cell 2cell 3
cell 1cell 2cell 3
cell 1cell 2cell 3
{| class="mycustomtable"
|-
|$ class="special"
|-
| =cell 1 || =cell 2 || =cell 3
|-
| ==cell 1 || cell 2 || ==cell 3
|-
| cell 1 || ==cell 2 || cell 3
|-
|}

Method 2 - define styles on the fly

You can add a style definition by using |$ style="CSSRULE" in a table. The definition can be used in any row below the definition by adding == before the contents of a cell.

Example default table:
Use this code to create the table:
cell 1cell 2cell 3
cell 1cell 2cell 3
cell 1cell 2cell 3
{|
|-
|$ style="background:springgreen"
|-
| *cell 1* || *cell 2* || *cell 3*
|-
| ==cell 1 || cell 2 || ==cell 3
|-
| cell 1 || ==cell 2 || cell 3
|-
|}
Example custom table:
Use this code to create the table:
cell 1cell 2cell 3
cell 1cell 2cell 3
cell 1cell 2cell 3
{| class="mycustomtable"
|-
|$ style="background:springgreen"
|-
| =cell 1 || =cell 2 || =cell 3
|-
| ==cell 1 || cell 2 || ==cell 3
|-
| cell 1 || ==cell 2 || cell 3
|-
|}

Multiple style definitions can be added to a table. The first definition can be used by adding == before the contents of a cell. The second definition can be used by adding === before the contents of a cell. The third definition can be used by adding ==== before the contents of any cell. And so on.

Example default table:
Use this code to create the table:
cell 1cell 2cell 3
cell 1cell 2cell 3
cell 1cell 2cell 3
{|
|-
|$ style="background:springgreen"
|-
|$ style="background:#ff00ff"
|-
|$ style="background:cyan"
|-
| *cell 1* || *cell 2* || *cell 3*
|-
| ==cell 1 || ===cell 2 || ====cell 3
|-
| ====cell 1 || =====cell 2 || ===cell 3
|-
|}
Example custom table:
Use this code to create the table:
cell 1cell 2cell 3
cell 1cell 2cell 3
cell 1cell 2cell 3
{| class="mycustomtable"
|-
|$ style="background:springgreen"
|-
|$ style="background:#ff00ff"
|-
|$ style="background:cyan"
|-
| =cell 1 || =cell 2 || =cell 3
|-
| ==cell 1 || ===cell 2 || ====cell 3
|-
| ====cell 1 || =====cell 2 || ===cell 3
|-
|}

These examples use a combination of method 1 and method 2. The first definition can be used by adding == before the contents of a cell. The second definition can be used by adding === before the contents of a cell. The third definition can be used by adding ==== before the contents of any cell. The fourth definition can be used by adding ===== before the contents of any cell. And so on.
