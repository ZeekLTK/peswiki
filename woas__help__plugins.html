All pages of the WoaS::Plugins:: namespace are WoaS plugins; plugins are executed when loading WoaS and remain active during WoaS usage. The WoaS project has plugin examples available for code syntax highlighting, MathML display, and charting; these are found in the Plugins directory of the download. One powerful use of plugins is to load a macro, or a set of macros, that will be available to all pages. The plugins directory also contains an Italian translation plug-in, though translation support is not yet fully capable in WoaS.

You can share WoaS plugins as WSIF files by exporting them.

External plugins

A WoaS plugin is just Javascript code, unless you create a plugin whose name starts with '@'. Such plugins are made up of external file references and a specific notation to force the browser into a specific inclusion mode.

Example:
path/to/mylib1.js = +*
path/to/mylib2.js = @*

The first resource, path/to/mylib1.js, will be included as an inline Javascript (e.g. like source code inside a <script > tag), while the second, path/to/mylib2.js, will be included using an external (<script> tag with proper 'src' attribute).

Inclusion operators

There are two inclusion operators which are valid for resource definitions:
  • + - which leads to inline inclusion
  • @ - which leads to external inclusion
The word which directly follows such inclusion operator can be any of the following supported tokens:
  • * catch-all, which means any browser/engine
  • ie Internet Explorer
  • firefox Mozilla Firefox (-)
  • opera Opera Internet Explorer
  • safari Apple Safari (-)
  • chrome Google Chrome
  • gecko Gecko engine (Mozilla)
  • webkit WebKit engine (Safari/Chrome)
  • presto Presto engine (Opera)
  • trident Trident engine (Internet Explorer)
Note: tokens marked with (-) do not support version matching

Now let's consider a more complex example:

path/to/mylib1.js = +* @gecko(1.9+)
path/to/mylib2.js = @*

path/to/mylib1.js will be loaded as inline for all browsers except those which have a Gecko engine with version 1.9 or above (i.e. Firefox 3.6 and above).
